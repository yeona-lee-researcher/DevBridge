## 1) 프로젝트 목표(결과물 중심)

- ✅ **기획 문서 완성**: 프로젝트 기획안 / 요구사항 명세서 / WBS / 수행일지

- ✅ **AI 자동 매칭/추천**: 사용자 요청/조건 기반 추천(프롬프트/룰/모델 연동 방식 포함)

- 🔐 **보안**: Spring Security로 인증/권한 분리 경험
- ☁️ **배포/운영**: Render/Koyeb/Aiven/GitHub Pages 등으로 “진짜 서비스” 운영 경험

# 🔄 운영 방식 (짧고 굵게!)

## 0) 협업 원칙 4가지

- 💬 **소통은 자주, 말은 짧게** (막히면 Discord로 바로 10분 안에 공유)
- 🧩 **작게 쪼개서 빠르게 완료** (큰 기능 = 작은 티켓 3~5개로 분해)
- 🔍 **PR은 반드시 리뷰 후 머지**
- 📌 **문서화는 회의록으로 작성, 의사결정 확정하며 진행**

## 1) Git & 개발 루틴(기본 세트)

- 브랜치: `main`(배포) / `dev`(통합) / `feature/기능명` / `fix/버그명`
- 커밋 규칙 예시: `feat: 매칭 API 추가` / `fix: 로그인 오류 수정`
- PR 규칙:
    - PR 설명에 **무엇을/왜/어떻게** + 스크린샷(가능하면) 첨부
    - 최소 **1명 리뷰 승인** 후 머지 → 포트폴리오 쓰고, 면접볼때 코드 협업 Tip*
    - 머지 후 **이슈 닫기 + 노션에 진행상태 체크**

## 2) 업무 관리(노션 + 이슈)

- 노션 보드: `Backlog → In Progress → Review → Done`
- 이슈 템플릿(짧게):
    - 목표 / 할 일 / 완료조건(DoD) / 참고자료 링크
- **Definition of Done(DoD)**
    - 기능 동작 확인 + 예외처리 1개 이상 + 간단 테스트/체크 + 문서/주석 최소

## 3) AI 도구 사용 원칙(효율 + 검증)

- ✅ ChatGPT/Claude/Gemini는 “초안/대안/리팩토링” 용도로 적극 사용
- ✅ 최종은 **팀이 이해하고 설명 가능한 코드/설계**만 채택
- ✅ 프롬프트/결정사항은 노션에 “짧게” 기록

# 📅 팀 정기모임 일정과 문화

## 1) 정기모임(추천 운영안)

- 🗓️ **주 5회 정기 팀미팅 시간 적극 활용 (각 60~90분)**
    - 특별하게 맡은 업무가 막힐 시에, 팀빌딩 시간 이후에 협의 인원들끼리 업무 추가 진행
- ☀️ **데일리 스크럼**
    - 서로의 기분과 마음을 실력상승과 같이 챙겨가기
    - 팀 분위기도 좋아야, 결과도 좋지않을까!

## 2) Daily 회의 구성

1. 🔥 **오늘의 회의안 목표 1~3개 확인 공유**
2. ✅ **완료된 것 데모/캡처 공유**
3. 🧱 **막힌 것 Top 1~2 해결(같이 보기)**
4. 🧾 **다음 액션 티켓화(담당/마감/완료조건)**

## 3) 팀 문화(귀엽지만 강력하게)

- 🧸 “가족 모드 ON” : 서로 말 예쁘게, 질문 환영, 비교 금지
- 🛠️ “프로 모드 ON” : 일정은 지키기, 약속은 기록하기, PR은 책임감 있게
- 🥳 작은 성취도 축하: “오늘 API 하나 뚫었다? 팀 승리👏”
- 🧯 갈등 룰: 감정X → 사실/근거/대안으로만 이야기(10분 내 결론)

## 🚀 Quick Start

### 1. DB 접속 (`src/main/resources/application.properties`)

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/devbridge_db?serverTimezone=Asia/Seoul
spring.datasource.username=YOUR_USERNAME
spring.datasource.password=YOUR_PASSWORD

# JWT (개발 기본값 OK, 배포 시 교체)
app.jwt.secret=${JWT_SECRET:dev-bridge-default-secret-key-change-in-production-please-32bytes}
app.jwt.ttl-hours=24

# Gemini API (AI 매칭용, 미설정 시 룰베이스 폴백)
gemini.api.key=${GEMINI_API_KEY:}
```

### 2. Backend (Spring Boot)

```bash
./gradlew bootRun         # 8080 포트
```

### 3. Frontend (React + Vite)

```bash
cd frontend
npm install               # 최초 1회
npm run dev               # 5173 포트 (API는 /api → 8080 프록시)
```

### 4. 접속

- 프론트: http://localhost:5173
- 백엔드 API: http://localhost:8080/api

## 🔐 Security

- **현재**: Spring Security 의존성 포함이지만 `build.gradle` 에서 주석 처리 (개발 편의). JWT 기반 경량 인증(`JwtAuthenticationFilter`) 만 활성.
- **프론트 토큰 관리**: `localStorage.accessToken` + Zustand `devbridge-storage` persist.
- **401 복구 시나리오**: Mypage 진입 같은 백그라운드 호출에선 redirect 안 하고 빈 응답 반환 → 사용자 입력 보존.
- **배포 전 체크리스트**:
  1. `build.gradle` 에서 `spring-boot-starter-security` 주석 해제
  2. `JWT_SECRET` 환경변수로 교체 (기본값 사용 금지)
  3. `@CrossOrigin` 하드코딩 제거 및 CORS 설정 분리
